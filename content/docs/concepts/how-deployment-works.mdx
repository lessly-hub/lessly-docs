A deploy is a build plus a release. The agent calls [`lessly_deploy`](/docs/reference/tools/lessly_deploy) with your git ref. Lessly runs the build in an isolated container, captures the output as an immutable artifact, then atomically routes traffic from the previous artifact to the new one. If the build fails, traffic never moves.
