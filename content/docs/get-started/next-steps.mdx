You have shipped one deploy. The rest of the docs teach you what else to ask for.

<Callout variant="note">
Your AI agent does the typing — these pages teach you what to ask for.
</Callout>

## Pick by intent

If you want to solve a specific problem, start with [Guides](/docs/guides) — recipes for [deploying a Next.js app](/docs/guides/deploy-nextjs), [setting up a custom domain](/docs/guides/custom-domain), and [rolling back a bad release](/docs/guides/rollback).

If you want to understand how Lessly works under the hood, read [Concepts](/docs/concepts) — short explanations of [how deployment works](/docs/concepts/how-deployment-works), the build system, and the security model.

If you need to look up a fact — a tool name, an environment variable, an error code — go straight to [Reference](/docs/reference). The MCP tool catalog lives there.

## A note on style

Every page assumes one thing: you have an AI agent open. The examples are prompts, not commands. The Reference pages name the tool the agent will call, so you can talk to your agent about Lessly using the same words the platform uses.
